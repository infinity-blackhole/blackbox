# Testing Strategy

## Test Layers

```
┌─────────────────────────────────────────────────────────┐
│  Integration Tests (tests/)                              │
│  Full service stacks, end-to-end flows                   │
├─────────────────────────────────────────────────────────┤
│  Component Tests (crate/tests/)                          │
│  Per-crate with mocked dependencies                      │
├─────────────────────────────────────────────────────────┤
│  Unit Tests (inline #[cfg(test)])                        │
│  Pure functions, serialization, comparison logic         │
└─────────────────────────────────────────────────────────┘
```

## Per-Crate Testing

### `core`

| Test | Method |
|------|--------|
| Config loading from TOML + env vars | Unit: parse known TOML, assert fields |
| Config env override | Unit: set env var, assert override |
| GameClock system vs frozen | Unit: `GameClock::system()` vs `GameClock::frozen(t)` |
| Error display messages | Unit: assert `to_string()` contains context |

### `master-data`

| Test | Method |
|------|--------|
| Decrypt known ciphertext | Unit: decrypt a known `.bin.e` fragment, assert plaintext |
| Parse TOC structure | Unit: decrypt small fixture, assert table names + offsets |
| Decompress LZ4 table | Unit: decompress known LZ4 blob, assert row count |
| Load full catalogs | Integration: load real `.bin.e`, assert table counts match Go output |
| Round-trip re-encrypt | Unit: decrypt → re-encrypt → decrypt, assert identical |
| Hot reload atomicity | Unit: spawn reader threads, reload, assert no torn reads |
| Mtime bump on reload | Unit: reload, assert file mtime changed |

**Test fixtures:** Small synthetic `.bin.e` files generated by the Go reference implementation.

### `schemas`

| Test | Method |
|------|--------|
| Deserialize positional array → struct | Unit: `rmp_serde::from_slice` on known bytes |
| All 200 structs compile | Compile-time: `cargo build` |
| Round-trip serialize/deserialize | Unit: struct → msgpack → struct, assert equality |
| Unknown field rejection | Unit: assert `deny_unknown_fields` rejects extra columns |

### `diff-engine`

| Test | Method |
|------|--------|
| `changed_tables` detects each field | Unit: mutate one field at a time, assert correct table name returned |
| `changed_tables` returns empty for identical | Unit: compare same state, assert empty vec |
| `compute_delta` produces correct JSON | Unit: before/after with known diff, assert JSON matches expected |
| Delete key computation | Unit: remove a key from map, assert `delete_keys_json` contains it |
| Update record computation | Unit: change a field, assert `update_records_json` has new value |
| No-op for unchanged rows | Unit: identical before/after, assert empty update/delete JSON |
| All 80+ table projectors produce valid JSON | Unit: project each table from a populated `UserState`, assert valid JSON |
| Composite key extraction | Unit: verify key fields for each table match `key_fields_for_table` |

**Property-based tests** (proptest):
- For any two `UserState` values, `changed_tables` returns a subset of all table names
- For any `UserState`, `compute_delta` with empty changed tables produces empty map
- Applying delta to before produces after (round-trip property)

### `store`

| Test | Method |
|------|--------|
| Create user → load user | Integration: insert via `SqliteStore`, load back, assert equality |
| Update user mutation | Integration: `update_user` with closure, assert mutation persisted |
| Session create → resolve | Integration: create session, resolve user_id, assert match |
| Session expiry | Integration: create with 1s ttl, sleep, assert resolution fails |
| Concurrent updates | Integration: spawn N tasks updating same user, assert no data loss |
| Migration idempotency | Integration: run migrations twice, assert no error |

**Test database:** In-memory SQLite (`sqlite::memory:`) for unit tests, temp file for integration tests.

### `grpc-server`

| Test | Method |
|------|--------|
| Each service method returns valid response | Integration: call each gRPC method via tonic test client |
| Session resolution from metadata | Integration: set `x-apb-session-key` header, assert correct user |
| Diff interceptor attaches delta | Integration: call mutating method, assert `DiffUserData` non-empty |
| Platform extraction | Integration: set platform header, assert context has correct platform |
| TimeSync trailer | Integration: call any method, assert `x-apb-response-datetime` trailer |
| Auth flow end-to-end | Integration: Register → Auth → GameStart → GetUserProfile |
| No-register mode | Integration: start with `--no-register`, assert RegisterUser fails |

**Test infrastructure:**
- `tonic::transport::Server::builder().add_service(...)` with in-memory channels
- `SqliteStore` with in-memory SQLite
- `MasterDataCatalogs` loaded from test fixture

### `cdn-server`

| Test | Method |
|------|--------|
| Serve list.bin | Integration: GET `/v2/.../list/0`, assert protobuf response |
| URL rewriting in list.bin | Integration: assert resource base URL replaced in response |
| Asset bundle serving | Integration: place file on disk, GET `unso-*` path, assert file content |
| MD5 validation | Integration: serve file with wrong MD5, assert 404 |
| Static HTML pages | Integration: GET `/information`, assert HTML response |
| Revision tracking | Integration: two requests from same addr, assert same revision |
| h2c support | Integration: HTTP/2 cleartext request, assert 200 |

### `auth-server`

| Test | Method |
|------|--------|
| Token validation | Integration: POST valid token, assert 200 with user info |
| Invalid token | Integration: POST invalid token, assert 401 |
| Username check | Integration: GET `/check-username`, assert availability |

### `admin`

| Test | Method |
|------|--------|
| Reload with valid token | Integration: POST with correct Bearer, assert 200 + catalogs reloaded |
| Reload with invalid token | Integration: POST with wrong Bearer, assert 401 |
| Reload without token env | Integration: start without `BLACKBOX_ADMIN_TOKEN`, assert port not bound |
| Constant-time comparison | Unit: verify timing-independent comparison (statistical test) |

## Integration Test Suite (`tests/`)

### Full Game Flow

```
1. Start all three services (grpc, cdn, auth) on ephemeral ports
2. RegisterUser → get userId
3. Auth → get sessionKey
4. GameStart with sessionKey
5. SetUserName, SetUserMessage
6. StartQuest → EndQuest (verify diff has IUserQuest)
7. GachaDraw (verify diff has IUserGacha, IUserMaterial)
8. GetUserProfile (verify no diff on read-only)
9. Verify final UserState matches expected
```

### Hot Reload Flow

```
1. Start grpc-server with initial .bin.e
2. Query a catalog value (e.g., quest end datetime)
3. Replace .bin.e with patched version
4. POST to admin reload endpoint
5. Query same catalog value — assert changed
6. Verify in-flight requests during reload see consistent state
```

### Concurrent Load

```
1. Start all services
2. Spawn N concurrent clients performing random game operations
3. Assert no panics, no data corruption
4. Assert all diffs are valid JSON
5. Assert final UserState is consistent (all mutations applied)
```

## Snapshot Testing

Use `insta` crate for snapshot tests:

- Master data table row counts (detect accidental schema changes)
- Diff output format (detect serialization changes)
- gRPC response shapes (detect proto drift)

```rust
#[test]
fn snapshot_table_row_counts() {
    let catalogs = load_test_catalogs();
    let counts: HashMap<&str, usize> = catalogs.table_row_counts();
    insta::assert_json_snapshot!("table_row_counts", counts);
}

#[test]
fn snapshot_diff_output() {
    let before = UserState::default();
    let mut after = UserState::default();
    after.profile.name = "TestPlayer".into();
    let diff = compute_delta(&before, &after, &changed_tables(&before, &after));
    insta::assert_json_snapshot!("diff_output", diff);
}
```
